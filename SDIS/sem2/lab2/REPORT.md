# Отчет по лабораторной работе

## Тема
Разработка оконного приложения с графическим пользовательским интерфейсом на языке Python по шаблону Model-View-Controller.

## Вариант 11
Информационная система хранит сведения о турнирах:

- название турнира;
- дата проведения;
- название вида спорта;
- ФИО победителя;
- размер призовых турнира;
- заработок победителя, который вычисляется автоматически как 60% от размера призовых.

## Назначение системы
Приложение предназначено для ввода, хранения, поиска, удаления, просмотра, импорта и экспорта записей о турнирах. В качестве постоянного хранилища используется база данных SQLite, для обмена файлами используется формат XML.

## Использованные технологии

- Python 3;
- PyQt5 для графического интерфейса;
- SQLite для хранения записей;
- `xml.dom.minidom` для записи XML через DOM;
- `xml.sax` для чтения XML через SAX.

## Архитектура MVC

### Model
Модель размещена в пакете `app/models` и описывает:

- структуру записи турнира;
- входные данные для создания записи;
- условия поиска и удаления;
- постраничный результат выборки;
- правила валидации и вычисления заработка победителя.

### Controller
Контроллер находится в `app/controllers/app_controller.py` и отвечает за:

- добавление записей;
- получение данных для главного окна;
- поиск и удаление по условиям варианта;
- импорт и экспорт XML;
- открытие и сохранение базы данных.

### View
Представление расположено в `app/views` и включает:

- главное окно;
- диалог добавления записи;
- диалог поиска с результатами в том же окне;
- диалог удаления;
- таблицу, дерево и виджет пагинации.

## Реализованные возможности

- одно главное окно и несколько дочерних диалогов;
- вызов команд через меню и дублирование на панели инструментов;
- ввод записи через отдельный диалог;
- автоматический расчет заработка победителя;
- хранение записей в базе данных SQLite;
- отображение записей в виде таблицы;
- альтернативное отображение текущей страницы записей в виде дерева;
- поиск по названию турнира, дате, виду спорта, фрагменту ФИО, диапазону призовых и диапазону заработка;
- удаление записей по отдельному диалогу условий;
- сообщение пользователю о количестве удаленных записей;
- экспорт текущего массива в XML через DOM;
- импорт массива из XML через SAX;
- загрузка и сохранение базы данных через стандартные диалоги;
- постраничный вывод в главном окне и в диалоге поиска.

## Структура базы данных

Используется таблица `tournaments` со следующими полями:

- `id` — целочисленный идентификатор;
- `tournament_name` — название турнира;
- `event_date` — дата проведения;
- `sport_name` — вид спорта;
- `winner_full_name` — ФИО победителя;
- `prize_amount` — размер призовых;
- `winner_earnings` — заработок победителя.

## Демонстрационные файлы

В каталоге `data` формируются следующие демонстрационные файлы:

- `demo_batch_1.xml` — 60 уникальных записей;
- `demo_batch_2.xml` — 60 уникальных записей;
- `demo_batch_1.db` — база данных с 60 записями;
- `demo_batch_2.db` — база данных с 60 записями;
- `tournaments.db` — основная база данных с суммарным набором данных.

## Сценарии демонстрации

1. Запустить приложение и показать главное окно с таблицей записей и постраничной навигацией.
2. Переключить отображение с таблицы на дерево и обратно.
3. Открыть диалог добавления, ввести новую запись и показать автоматический расчет заработка победителя.
4. Открыть диалог поиска, выполнить поиск по виду спорта и по фрагменту ФИО победителя, показать результаты с пагинацией.
5. Открыть диалог удаления, задать условия и показать сообщение о количестве удаленных записей.
6. Экспортировать текущие данные в XML-файл.
7. Импортировать данные из одного из подготовленных XML-файлов.
8. Открыть другую базу данных через стандартный диалог и показать загрузку другого набора записей.
9. Сохранить текущую базу данных под новым именем через стандартный диалог.

## Запуск проекта

1. Установить зависимости:

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

2. Сформировать демонстрационные данные:

```bash
.venv/bin/python scripts/generate_demo_data.py
```

3. Запустить приложение:

```bash
.venv/bin/python main.py
```
